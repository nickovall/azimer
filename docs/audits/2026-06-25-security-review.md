## Закрыто в этой сессии

### 1. `kp_sessions` — утечка PII (было: HTTP 200 с данными под анон-ключом)
Анон-ключ (он в JS сайта у любого) выкачивал все сессии Telegram-бота: `chat_id`,
имена, телефоны, черновики КП. Причина: миграция `harden_public_access`
применилась в прод не полностью.
**Фикс:** `enable RLS` + `revoke all from anon, authenticated` (Management API).
**Проверено:** внешняя проба теперь `HTTP 401 permission denied`; в БД
`relrowsecurity=true`, грантов anon/authenticated нет. Бот/админка под
`service_role` (bypass RLS) — не затронуты.

### 2. Спам/SMS-абуз — рейт-лимит на вставку лида
Вставка в `leads` разрешена анон-ключом (форма). Триггер `leads_notify_telegram`
(AFTER INSERT) слал SMS через smsc.ru **без лимита** → сжигание баланса +
SMS-бомбинг чужих номеров + флуд Telegram. Клиентский антибот обходится прямым
запросом к PostgREST.
**Фикс:** BEFORE INSERT триггер `a_leads_rate_limit` (срабатывает раньше notify).
Троттлит только роли `anon`/`authenticated` (бот/админка/миграции под
`service_role` — мимо). Потолки: ≥3 на номер/10 мин, ≥12/час, ≥40/сутки
(`source_channel='site'`). При превышении вставка отклоняется → ни SMS, ни Telegram.
**Проверено:** триггер установлен BEFORE INSERT; легит-объём 0 в окнах (запас огромный).
**Не лечит полностью:** во время атаки на потолке легит-заявка тоже отклоняется.
**Настоящее лечение (follow-up):** Cloudflare Turnstile + вставка за Edge Function
— боты отсекаются на входе, лимиты не нужны, лиды не теряются.

## Открыто

### 3. `/kp` без подписи
`app/kp/page.tsx` рендерит КП из `#data=<base64>` без проверки подлинности →
ссылку можно переписать (цены/текст/заказчик) и показать на azimer.ru как
«официальное» КП. Не взлом сервера; риск — споры/репутация/фейк третьему лицу.
**Фикс:** HMAC-подпись серверных ссылок (бот/админка; инфраструктура HMAC уже
есть в `admin-api`), `/kp` проверяет подпись; клиентские превью помечать
«предварительный расчёт, не оферта».

### 5. `smooth-task` = ЖИВОЙ Telegram-бот (деплой-дрейф)
Опознан по исходнику (скачан через `supabase functions download`): это
**АЗИМЕР Telegram bot webhook handler** — старая версия, залитая через Dashboard
(слаг авто-сгенерён, визард вшит инлайном, цены захардкожены, без каталога,
обновлён 31.05). **Telegram-вебхук указывает именно на него** (проверено
`getWebhookInfo` 25.06: url = `.../functions/v1/smooth-task`, `pending=0`).

Из этого следует **деплой-дрейф**: поддерживаемый в репо `telegram-bot` (свежий,
v6) вебхук-трафик НЕ получает — доработки бота в проде не работают. Плюс исходник
боевого бота **не в репозитории** (бэкап сохранён: `B:/dbtest/smooth-task-backup-2026-06-25.ts`).

**НЕ удалять** `smooth-task` — это сломает бота Азамата. Правильный фикс
(отдельная аккуратная задача): закоммитить боевой исходник в репо; переключить
вебхук на `telegram-bot`, проверить тестовым апдейтом + что env
(`WEBHOOK_SECRET`, `ADMIN_CHAT_IDS`) на месте и Азамат получает карточки; только
потом ретайрить `smooth-task`.
