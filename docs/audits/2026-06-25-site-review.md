## 1. ВЕРДИКТ ПО САЙТУ

**Сайт живой, работает в проде, принимает заявки. Дизайн — выше среднего по рынку
коммерческого строительства. Два реальных пробела: (а) доказательная база контента
и (б) хардненинг бэкенда перед масштабированием платного трафика.**

Оценка по направлениям:

| Направление | Оценка | Комментарий |
|---|---|---|
| Дизайн / UX (фронт) | 8 / 10 | Цельная инженерно-премиальная система, отстраивает от демпинг-сегмента |
| Захват лидов / конверсия (инфра) | 7 / 10 | Два пути (быстрый контакт + мастер расчёта), mobile sticky CTA, КП/PDF, бот |
| Доказательная база (числа, кейсы, команда) | 4 / 10 | Главный маркетинговый недобор — нет реального трек-рекорда на витрине |
| SEO / тех-гигиена фронта | 8 / 10 | sitemap/robots/OG/meta/favicons, typecheck чистый, статэкспорт |
| Безопасность / целостность бэкенда | 🟢 в осн. закрыто | Ре-верифицировано 25.06 (`2026-06-25-security-review.md`): admin-api/бот/RLS закрыты; найдена и устранена живая утечка `kp_sessions`; SMS-абуз ограничен. Открыто: подпись `/kp`, функция `smooth-task` |

**Итог по готовности:**
- Для **органического трафика** (SEO, сарафан, 2GIS) — готов и уже работает.
- Перед **масштабированием платного трафика и ростом потока лидов** — закрыть
  P0 безопасности/целостности (§3) и добавить доказательную базу (§3, контент).
- Дизайн переделывать **не нужно** — недобор не в эстетике, а в контенте и хардненинге.

## 2. ТЕХНИЧЕСКАЯ ДОКУМЕНТАЦИЯ

### 2.1 Стек

| Слой | Технология |
|---|---|
| Фронт | Next.js 16 (`output: "export"`, `trailingSlash: true`, `images.unoptimized`) |
| UI | React 19, Tailwind CSS 4, framer-motion, lenis (smooth scroll) |
| Хостинг | GitLab Pages + Cloudflare DNS (домен azimer.ru) |
| БД / бэкенд | Supabase (Postgres + RLS) |
| Edge Functions | `admin-api`, `telegram-bot`, `catalog-admin` (Deno) |
| Генерация КП | `@react-pdf/renderer` (PDF в фирменном стиле) |
| Уведомления | Telegram bot, SMSC.ru (SMS), Resend (email) |
| Аналитика | Yandex.Metrika |

### 2.2 Архитектура (поток данных)

```text
Публичные формы / Мастер расчёта (/estimate, /contacts, /partners)
        | submitLead()  (lib/supabase.ts)
        v
   public.leads  (Supabase Postgres, RLS)
        |  DB-триггер (фильтр + нотификации)
        +--> SMS (SMSC)            --> клиент / Азамат
        +--> Telegram bot          --> карточка заявки, статусы, /kp /leads
        |
        v
   Админ-консоль /console-x9p4m2/*  --(service-role)-->  admin-api (Edge Function)
        |  лиды, статусы, документы (Google Drive URL), КП, шаблоны, рассылка
        v
   Калькулятор: lib/pricing -> lib/calculator (engine, classifier, regions,
   consumption, geometry, catalog.generated)  <- build-catalog из Supabase
```

### 2.3 Карта маршрутов

**Публичные:** `/` · `/services` · `/projects` · `/about` · `/contacts` ·
`/estimate` (мастер расчёта) · `/estimate-project` (форма по готовому проекту) ·
`/partners` · `/privacy` · `/thanks` · `/kp` (просмотр КП по `#data`).

**Админ (gate-key + пароль):** `/console-x9p4m2/` · `/leads` (+ `/view`, `/new`,
`/kp`) · `/catalog` · `/templates` · `/compose`.

### 2.4 Структура репозитория (ключевое)

```text
azimer-site/
├── app/                  страницы (public + console-x9p4m2)
├── components/           Hero, секции, формы, estimate/*, admin/*, ui/*
├── lib/
│   ├── content.ts        ВЕСЬ контент сайта (тексты, фото-ссылки, факты)
│   ├── pricing.ts        обёртка калькулятора для мастера расчёта
│   ├── calculator/       инженерный движок (engine, classifier, regions,
│   │                     consumption, geometry, catalog.generated.ts)
│   ├── supabase.ts       публичная отправка заявок
│   └── admin-api.ts      клиент админ Edge Function
├── supabase/
│   ├── migrations/       схема, RLS, триггеры
│   └── functions/        admin-api, telegram-bot, catalog-admin
├── scripts/              build-catalog, build-qr, build-bot-bundle
├── public/photos/        реальные фото объектов
└── docs/                 аудиты, llm-context, roadmap
```

### 2.5 Деплой

- **Сайт + бот-бандл:** `git push gitlab main` → GitLab CI → `next build` →
  GitLab Pages. `prebuild` генерит каталог из Supabase (нужны
  `NEXT_PUBLIC_SUPABASE_*` в CI/env) и QR.
- **Edge Functions** (`admin-api`, `telegram-bot`) — деплоятся **вручную**
  через `supabase functions deploy ...` (НЕ авто из CI — источник дрейфа, см. §3).
- **Кэш:** Cloudflare + браузер агрессивные. Картинки с теми же именами
  требуют hard-refresh/инкогнито; новые имена файлов подтягиваются сразу.

### 2.6 Переменные окружения (имена; значения — в `B:\dbtest\`)

- **Публичные (build):** `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`,
  `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`, `NEXT_PUBLIC_ADMIN_GATE_KEY`.
- **Edge Functions:** `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `ADMIN_PASSWORD`,
  `ADMIN_SESSION_SECRET`, `SITE_URL`, `ADMIN_ALLOWED_ORIGINS`, `ACCOUNTANT_EMAIL/PHONE`,
  `RESEND_API_KEY`, `SMSC_LOGIN/PASSWORD`, Telegram token + webhook secret.

### 2.7 Текущее состояние тех-долга (по аудиту 05.06.2026 — требует ре-верификации)

> Полные таблицы — в `2026-06-05-technical-audit.md`. Ниже сжато; часть могла быть
> закрыта в волнах после 05.06 (Wave 1 docflow, поставщики цен 15.06).
>
> **Ре-верифицировано 25.06** (`2026-06-25-security-review.md`): admin-api, бот,
> RLS на leads/документах — закрыты; найдена и устранена живая утечка `kp_sessions`;
> SMS-абуз ограничен рейт-лимитом. Открыто: подпись `/kp`, функция `smooth-task`.

**P0 (блокеры перед платным трафиком):** RLS на `kp_sessions`; ограничение
server-owned полей в public insert `leads`; хардненинг `admin-api` (короткоживущий
токен вместо пароля в теле, узкий CORS); бот — fail-closed на пустом
`ALLOWED_CHAT_IDS` + обязательный webhook secret; целостность `/kp#data=`
(сейчас неподписанный base64).

**P1:** формы показывают success даже при ошибке вставки (теряются лиды);
тесты калькулятора вне репо/CI; невалидный ввод калькулятора (нули/отрицательные/
огромные объекты); снапшот цены в КП; CI не деплоит admin-api; нет
typecheck/lint в CI gate.

## 3. ПЛАН УЛУЧШЕНИЯ (приоритизированный)

### 🔴 P0 — до масштабирования трафика
1. **Целостность лидов в формах** — показывать ошибку, success только после
   подтверждённой вставки (иначе горячие лиды теряются молча). Затрагивает
   `ContactForm`, `ProjectForm`, `PartnerForm`, `RaschetWizard`.
2. **RLS / public insert** — закрыть `kp_sessions`, ограничить принимаемые поля
   `leads` (server-owned → дефолты в БД).
3. **Хардненинг `admin-api` и бота** — токен вместо пароля в теле, узкий CORS,
   fail-closed бот, webhook secret обязателен в проде.
4. **Подпись/непереписываемость `/kp` payload**.

### 🟡 P1 — качество и устойчивость
5. **Доказательная база контента** (главный маркетинговый рычаг):
   - реальные числа в Stats (объектов, м², «с 2023») — ждёт портфолио;
   - 3–4 кейса на `/projects` с фактами (регион/площадь/срок/состав);
   - блок о команде / Азамате.
6. **CI-гейты:** `typecheck` + `test:calculator` в пайплайн; авто- или
   документированный деплой `admin-api`; снять конкурирующий GitHub Pages workflow.
7. **Калькулятор:** отвергать невалидный ввод, гейтить «инженер обязателен»
   (вечная мерзлота/сейсмика/большие пролёты/тяжёлые краны).
8. **Reveal** — контент видим по умолчанию (сейчас `opacity:0` до JS — риск
   пустых экранов и для SEO-краулеров). `components/ui/Reveal.tsx`.

### 🟢 P2 — гигиена / усиление
9. Блок FAQ / возражения (цена «от», сроки, оплата, гарантия).
10. Карта/схема проезда на `/contacts`.
11. B2B-блок «Для бизнеса»: апгрейды КС-2/КС-3, тендеры, гарантия — по
    подтверждению владельца.
12. Снапшот цены в КП; аудит-лог админ-действий; уникальность активной цены
    в `catalog_items`.

### Что НЕ делать
- Возвращать благодарственные письма/отзывы на сайт (решение владельца).
- Переписывать калькулятор с нуля.
- Большой редизайн — не требуется.

## 4. ДИЗАЙН

### 4.1 Дизайн-система

| Токен | Значение |
|---|---|
| Цвета | графит `#1D1C1C`/`#111`, оранжевый `#ED6629`, светлый `#F6F7F6`, линия `#D9D9D6` |
| Шрифты | Inter (текст) + IBM Plex Mono (технические лейблы, цифры) |
| Фактуры | blueprint-grid и blueprint-paper (чертёжная сетка) — поддерживает инженерный бренд |
| Моушн | framer-motion, дозирован; `prefers-reduced-motion` отключает |

### 4.2 Сильные стороны
- Цельный инженерно-премиальный язык — отстраивает от типового «ангары дёшево»,
  совпадает с позиционированием (премиум, не демпинг).
- Сильный Hero: реальное фото монтажа, чёткий оффер «под ключ», два CTA.
- Аккуратная типографика/сетки/hover; шапка с гистерезисом против дёрганья на iOS.
- Конверсионная инфраструктура: быстрый контакт + мастер расчёта + mobile sticky CTA.
- Доверие закрыто базово: ИНН/ОГРН/адрес, прямой телефон/VK, B2B-блок с НДС.

### 4.3 Слабые стороны
- **Числа доверия описывают процесс, а не масштаб** (3 типа каркаса / 8 этапов /
  24 ч) — нет реального трек-рекорда. Главная дыра.
- **/projects — галерея, а не кейсы** — фото без фактов (регион/площадь/срок).
- **Нет «человеческого» доказательства** — фото бригады/Азамата (частично закрыто
  кадрами «в каске», но блока о команде нет).
- **Reveal** гейтит контент анимацией (тех-риск, см. §3.8).

### 4.4 Сделано в этой сессии (24–25.06)
- Заменены/оптимизированы фото: галерея «Каркас и контур», свап
  «Металлокаркас ↔ Сварочные», кровля «в каске», склад вместо неба,
  карточки `/projects` «Производство» и «Технические сооружения».
- Все новые фото пересжаты (PNG ~2 МБ → JPG 120–190 КБ).
- Добавлен B2B-блок «Для бизнеса» на `/services` (НДС, документы, безнал).
- Удалены мусорные файлы (дефолтные Next-SVG, осиротевший `obj-sklad.jpg`).

### 4.5 Дизайн-рекомендации (по убыванию отдачи/усилия)
1. Реальные числа в Stats (контент) — максимальный эффект.
2. 3–4 настоящих кейса с фактами на `/projects`.
3. Блок о команде/основателе + фото производства с людьми.
4. Блок гарантии/FAQ — снять возражения визуально и текстом.
5. Фикс Reveal — контент видим по умолчанию.
