## 🔴 BUGs — что починить (3 теста, 1 файл)

### BUG #1: Расчёт количества свай через геометрию, не нагрузку

**Файл**: `azimer-site/lib/calculator/modules/foundation.ts:46-51`

**Текущий код**:
```ts
const L = Math.max(input.length, input.width);
const W = Math.min(input.length, input.width);
const cols = Math.max(2, ceil(L / pileStep) + 1);
const rows = Math.max(2, ceil(W / pileStep) + 1);
let piles = ceil(cols * rows * 1.30);
```

**Проблема**: cols/rows растут сверхбыстро для маленьких зданий из-за `max(2, …) + 1`. Для здания 7.44×4.60 при step=4 → cols=3, rows=3 → 9 точек × 1.30 = **12 свай**. На массу здания 3-5 т (Белый Куб) это 4× избыток.

**Эталон** (СП 24.13330 + svaisad.ru):
- Свая 108×2500 несущая способность 5 т
- Жилой модуль 34 м² с лёгким каркасом: масса 5-15 т → 2-3 сваи по нагрузке + запас на распределение → **4-6 свай по периметру обвязки**
- Здание 12×24 metal warm: 50-80 т → 10-16 свай по нагрузке + запас → **15-20 свай**

**Предлагаемая правка**: считать через **массу здания** + распределение, а не через сетку точек.

```ts
// Грубая оценка массы здания (т)
const frameWeightKg = computeFrameMassKg(input, geo);  // уже есть в frame.ts
const cladWeightKg = geo.wallArea * (isCold ? 8 : 22); // профлист ~8 кг/м², сэндвич ~22
const roofWeightKg = geo.roofArea * (isCold ? 8 : 22);
const totalKg = frameWeightKg + cladWeightKg + roofWeightKg + geo.floorArea * 50; // 50 кг/м² нагрузка
// 1 свая Ø108 = 5т, Ø133 = 8т
const pileCapacityKg = isHeavyLoad ? 8000 : 5000;
const pilesByLoad = ceil(totalKg / pileCapacityKg);
// Минимум по геометрии — углы + периметр (1 свая на каждые 3 м периметра + углы)
const pilesByGeometry = ceil(geo.perimeter / 3) + 4;
let piles = Math.max(pilesByLoad, pilesByGeometry);
```

**Эффект**: Белый Куб 12→6 свай (−50%), 12×24 теплый 37→18 (−51%), 12×24 холодный 20→12 (−40%). Реально приближается к рынку.

### BUG #2: `heavyLoad` назначается по типу каркаса, а не по нагрузке

**Файл**: `foundation.ts:36`

**Текущий код**:
```ts
const isHeavyLoad = (input.craneCapacityT ?? 0) > 0 || (input.frame === "metal" && !isCold);
```

**Проблема**: ЛЮБОЙ тёплый металлокаркас → heavyLoad → Ø133×3000 (9 500 ₽/шт) вместо Ø108×2500 (6 500 ₽/шт). Жилой модуль 34 м² с лёгким каркасом 3-5 т получает сваи рассчитанные на 30+ тонн.

**Эталон**: диаметр сваи выбирается по нагрузке:
- Ø89: до 3 т (садовые постройки, бытовки)
- Ø108: 3-5 т (каркасные дома, лёгкие здания)
- Ø133: 5-8 т (тяжёлые здания, с подвалом, многоэтажные)
- Ø159+: 8+ т (промышленные, мерзлота)

**Предлагаемая правка**:
```ts
// isHeavyLoad по нагрузке на 1 сваю, не по типу каркаса
const loadPerPileKg = totalKg / pilesByLoad;
const isHeavyLoad = loadPerPileKg > 5000 || (input.craneCapacityT ?? 0) >= 5;
```

**Эффект**: лёгкие жилые/коммерческие здания получают Ø108 вместо Ø133. Экономия на единичной свае ~3 000 ₽.

### BUG #3: Холодные здания тоже завышают свайную сетку

**Файл**: `foundation.ts:41-44` (та же формула, что #1, но с pileStep=6)

**Текущий код для cold**: `pileStep = 6.0`. Для 12×24 cold → cols=⌈24/6⌉+1=5, rows=⌈12/6⌉+1=3 → 5×3×1.30 = 20 свай.

**Проблема**: 20 свай Ø108 на холодный ангар 12×24 m² (масса ~10-15 т) — нагрузка на сваю 0.5-0.75 т (свая выдерживает 5). 10-15× недонагрузка. Реальный рынок — 10-12 свай.

**Решение**: то же что BUG #1 — расчёт по нагрузке.

## 🟡 LIMITATIONS (приемлемые упрощения, документировать)

Эти проблемы тесты **не** пометили как fail (engine работает по формуле), но формула — **упрощённая эмпирика**, не норматив. Это `BY_DESIGN` решения автора engine. Можно улучшать, но не блокирует точность.

### LIM #1: ROOF_SLOPE_FACTOR = 1.05 жёсткий для всех типов кровли

**Файл**: `geometry.ts:16`

```ts
const ROOF_SLOPE_FACTOR = 1.05;  // 5% запас на уклон
```

**Реальность**: уклон зависит от типа покрытия:
- Сэндвич: 5-10° → cos = 0.985-0.996 → +0.4-1.5% к площади
- Профлист: 12-20° → +2-6%
- Металлочерепица: 20-30° → +6-15%

**Эффект**: для сэндвич-кровли расход панелей завышен на 3-5%. Не критично. Если правим — добавить логику в `geometry.ts` чтобы factor зависел от `roofing`.

### LIM #2: wall_area без вычета проёмов

**Файл**: `walls.ts:13` + `geometry.ts:26`

```ts
const wallArea = perimeter * H;  // без вычета окон/дверей/ворот
```

**Реальность**: для здания 24×12×6 с воротами 4×4 + 5 окнами 1.5×2 + дверью = ~33 м² проёмов. Engine считает сэндвич на всю стену 432 м², включая проёмы.

**Эффект**: завышение материала стен на 5-15% в зависимости от количества проёмов. Поэтому консервативная оценка вверх.

**Фикс если хочешь**: вычитать `openingsArea` из `wallArea` перед расчётом панелей и крепежа.

---

### LIM #3: edge_trim_factor = 12% от площади стен

**Файл**: `consumption.ts:22` (SANDWICH_NORMS)

```ts
edge_trim_factor: 0.12,  // 12% площади стен в фасонных элементах
```

**Реальность**: фасонные элементы — это нащельники по углам, парапеты, угловые планки. Для типового 12×24 это ~4-6 м² (4 угла × 6 м × 0.3 м ширины планки). 12% от 432 м² = 52 м² — **в 8-10× больше реальности**.

**Эффект**: фасонка в расчёте 52 м² × 1056 ₽ = 55K ₽ vs реально 6 м² × 1056 = 6K ₽ → **+49K ₽** в каждый КП.

**Альтернативная формула**:
```ts
edge_trim_factor: 0.015,  // ~1.5% (по периметру углов × ширина планки)
// или
const trimM2 = (geo.perimeter + ridgeLength + openingsPerim) * 0.3;
```

### LIM #4: Холодное здание × 0.60 завышает реальный холодный ангар

**Файл**: `frame.ts:94-96`

```ts
if (detectInsulation(input) === "cold") {
  kgPerM2 *= 0.60;  // -40% к расходу металла
}
```

**Реальность**: рыночный холодный ангар 12×24 имеет 3-5 т металла (10-17 кг/м²). Наш расчёт после ×0.60 от warm 52.6 → 31.5 кг/м² × 288 = **9 т**. Это близко к "облегчённому тёплому", а не настоящему холодному.

**Альтернативная формула**: cold ×0.40 (вместо 0.60) даст 21 кг/м² ≈ рынок.

**Замечание**: это **намеренный консерватизм** автора. Если хочешь подтянуть к рынку — меняй с 0.60 на 0.45-0.50.

### LIM #5: Sealant tubes по странной формуле `P × 2 / 8`

**Файл**: `walls.ts:72-84`

```ts
const sealantQty = ceil(P * 2 * SANDWICH_NORMS.sealant_tubes_per_m);  // 1/8
const foamQty    = ceil(P * 2 * SANDWICH_NORMS.foam_cans_per_m);     // 1/8
```

**Что это значит**: на здании 12×24 P=72 → P*2=144 → 144/8 = **18 туб герметика + 18 баллонов пены**.

**Реальность**: длина швов = вертикальные швы по 1 на 1.19 м ширины панели × высоту 6 м. Для P=72 → ~60 вертикальных швов × 6 м = 360 м.п. шва. По 1 туба на 8 м.п. → 45 туб. **Engine считает 18, реальность 45** → недосчитывает в 2.5×.

Хотя цена тубы 264 ₽ — разница 7 K ₽. Не критично, но формула странная.

### LIM #6: Roof flashings = P × 0.15

**Файл**: `roof.ts:74`

```ts
const trimQty = round(P * 0.15, 0);  // 15% периметра
```

Для P=72 → 11 м.п. нащельников. Реальная парапетная планка идёт по всему периметру (72 м), а ендова + конёк отдельно. 15% — занижение.
